Civic Graph

Civic Graph is an open-source and crowdsourced guide for civic tech’s developing network of connections. During my time on the project, I built out an analytics dashboard, restructured the codebase to add some exciting new features, implemented a scalable check-in system for events, and restructured the application architecture. This has been particularly exciting because of my interest in collecting, analyzing, and visualizing information in a way that's open, accessible, and progressive. Because civic graph is open and crowdsourced, it displays a unique knowledge base of its own. To be able to administer a digital space that offers transparency and helps define ‘civic tech' has been a uniquely rewarding experience.

Civic Graph's UI, API, and Check-In repositories are on GitHub. Civic Graph can be found at civicgraph.io.

My responsibilities on the project have included the following:

Infrastructure Cleanup
  • Lint codebase (Recurring - every 2-3 weeks)
  • Refactor codebase by splitting application into respective directives & services and setting scope variables (best practice)
  • Reorganize file structure according to Angular style guide recommendations
  • Protect controller against dependency issues when minified
  • Implement gulp.js (automate workflow, speed up compile time, better structure js files)
  • Add global util service for mobile graph (better event-handling)
API Cleanup
  • Wrote & published API documentation
  • Edited API to visualize node entries from check-in application
  • Decoupled UI from API (best practice for scale, deploying UI independently for check in)
UI Enhancements
  • Implemented slider & number of connections
  • Built & deployed Power BI analytics dashboard
  • Cleaned nav and relocated search bar
Little Things

Check-In Application
